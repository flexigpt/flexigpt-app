## Tips for stronger prompts

- Start with the exact outcome you want.
- Specify the format you want back, such as bullets, table, JSON, or draft copy.
- Include constraints like audience, tone, word count, or programming language.
- If the task is large, ask for a short plan first and then continue step by step.

## When to use attachments

- Attach source files instead of pasting very large content when that is easier.
- Add supporting documents when you want the answer grounded in those files.
- Re-check attachments before sending if they contain sensitive or irrelevant material.

## Tuning model behavior

- Use lower **temperature** for repeatable, factual, or structured work.
- Use higher temperature for brainstorming, ideation, or alternative phrasings.
- If the model supports **reasoning**, increase it when you want deeper chain-of-thought style work, but expect slower or more expensive requests depending on the provider.

## When answers look wrong

- Simplify the prompt.
- Remove conflicting instructions.
- Ask the model to explain assumptions.
- Provide a small example of the output you want.
- Try a different model or preset if the current one is a poor fit.
